## Краткие теоретические сведения

### Файловая структура системы LINUX

В операционной системе **LINUX** файлами считаются **обычные файлы**, **каталоги**, а также **специальные файлы**, соответствующие периферийным устройствам (каждое устройство представляется в виде файла). Доступ ко всем файлам однотипный, в том числе и к файлам периферийных устройств. Такой подход обеспечивает независимость программы пользователя от особенностей ввода/вывода на конкретное внешнее устройство.

Файловая структура LINUX имеет иерархическую древовидную структуру. В **корневом каталоге** (`/`) размещаются другие каталоги и файлы, включая 5 основных каталогов:

| Каталог | Назначение |
|---------|-------------|
| `/bin` | большинство выполняемых командных программ и shell-процедур |
| `/tmp` | временные файлы |
| `/usr` | каталоги пользователей (условное обозначение) |
| `/etc` | преимущественно административные утилиты и файлы |
| `/dev` | специальные файлы, представляющие периферийные устройства |

> **Примечание:** при добавлении периферийного устройства в каталог `/dev` должен быть добавлен соответствующий файл (черта `/` означает принадлежность корневому каталогу).

**Текущий каталог** — это каталог, в котором в данный момент находится пользователь. При наличии прав доступа пользователь может перейти после входа в систему в другой каталог.

- Текущий каталог обозначается точкой ( **.** )
- Родительский каталог обозначается двумя точками ( **..** )

**Полное имя файла** может включать имена каталогов, включая корневой, разделенных косой чертой, например:  
`/home/student/file.txt`

> Первая косая черта обозначает корневой каталог, и поиск файла будет начинаться с него, затем в каталоге `home`, затем в каталоге `student`.

Один файл можно сделать принадлежащим нескольким каталогам. Для этого используется команда **ln** (link):

```bash
ln <имя файла 1> <имя файла 2>
```

# Ответы на вопросы по ОС Linux

## 1. Что считается файлами в OC Linux?

В философии Linux (и UNIX в целом) используется принцип: **«Всё есть файл»**.

Это означает, что файлом считается не только обычный документ или текст (Regular file), но и:
*   **Каталоги (Directories)** — специальные файлы, содержащие имена других файлов.
*   **Символьные и блочные устройства (Device files)** — например, `/dev/sda` (жесткий диск) или `/dev/tty` (терминал).
*   **Каналы (Pipes / FIFO)** — для передачи данных между процессами.
*   **Сокеты (Sockets)** — для межпроцессного взаимодействия.
*   **Символические ссылки (Symlinks)** — указатели на другие файлы.

Файл характеризуется: **именем**, **inode** (структура, хранящая метаданные: права, владельца, размер, расположение на диске) и **содержимым**.

## 2. Объясните назначение связей с файлами и способы их создания.

Связи (Links) позволяют обращаться к одному и тому же физическому содержимому диска (inode) по разным именам.

### Жесткие связи (Hard links)
*   **Назначение:** Создание дополнительного имени для существующего файла. По сути, это еще одна запись в каталоге, указывающая на тот же inode.
*   **Особенности:** Нельзя создать для каталогов (чтобы избежать циклов) и на разные файловые системы (т.к. inode уникален в пределах одной ФС). При удалении файла данные стираются только когда счетчик ссылок станет 0.
*   **Создание:** `ln target_file link_name`

### Символические связи (Symlinks / Soft links)
*   **Назначение:** Специальный файл, который содержит *путь* к другому файлу или каталогу (похоже на ярлык в Windows).
*   **Особенности:** Можно ссылаться на каталоги и на другие ФС. Если исходный файл удалить, ссылка "сломается" (повиснет).
*   **Создание:** `ln -s target_path link_name`

## 3. Что определяет атрибуты файлов и каким образом их можно просмотреть и изменить?

Атрибуты файлов хранятся в **inode** и определяются следующими параметрами:

1.  **Тип файла** (`-` (файл), `d` (каталог), `l` (ссылка) и т.д.).
2.  **Права доступа (Mode):** r (чтение), w (запись), x (исполнение) для владельца, группы и остальных.
3.  **Владелец (Owner / UID)** и **Группа (Group / GID)**.
4.  **Временные метки:** Последнее изменение содержимого (mtime), последнее изменение атрибутов (ctime), последний доступ (atime).

### Просмотр:
*   `ls -l` — подробный список (права, владелец, группа, размер, дата).
*   `stat <файл>` — максимально подробный вывод метаданных.

### Изменение:
*   **Права доступа:** `chmod` (например, `chmod 755 file` или `chmod u+x file`).
*   **Владелец:** `chown` (например, `chown user:group file`).
*   **Группа:** `chgrp` (например, `chgrp group file`).

## 4. Какие методы создания и удаления файлов, каталогов Вы знаете?

### Создание:

| Объект | Команда | Пример |
| :--- | :--- | :--- |
| **Пустой файл** | `touch` | `touch file.txt` |
| **Файл с данными** | Редирект (`>`), `cat`, `echo` | `echo "text" > file.txt` |
| **Файл (через редактор)** | `nano`, `vim`, `gedit` | `nano file.txt` |
| **Каталог** | `mkdir` | `mkdir my_dir` (создать вложенные: `mkdir -p path/to/dir`) |

### Удаление:

| Объект | Команда | Примечание |
| :--- | :--- | :--- |
| **Файл** | `rm` | `rm file.txt` |
| **Пустой каталог** | `rmdir` | `rmdir my_dir` |
| **Каталог с содержимым** | `rm -r` (или `-rf`) | `rm -r my_dir` (опасно, `-f` для принудительного удаления) |

> **Важно:** В Linux нет "Корзины" при использовании `rm`. Удаление через `rm` — это потеря ссылки на inode; восстановление данных крайне сложно без специального ПО.
