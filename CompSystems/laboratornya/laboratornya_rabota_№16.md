## Теоретические сведения

### Что такое журнал событий?

Даже когда пользователь ПК не совершает никаких действий, операционная система продолжает считывать и записывать множество данных. Наиболее важные события отслеживаются и автоматически записываются в особый лог, который в Windows называется **Журналом событий**.

**Основное предназначение журнала событий** — сбор данных, которые могут пригодиться при устранении неисправностей в работе системы, программного обеспечения и оборудования.

> **Важно:** В журнал заносятся не только ошибки, но также предупреждения и удачные операции (установка программы, подключение к сети и т.д.).

### Как открыть журнал событий?

В системе предусмотрена специальная штатная утилита **eventvwr**. Запустить её можно:

- **Способ 1:** `Панель управления` → `Администрирование` → `Просмотр событий`
- **Способ 2:** Нажать `Win + R`, ввести `eventvwr.msc` и нажать `Enter`

**Структура окна утилиты:**

| Область | Содержание |
| :--- | :--- |
| **Левая колонка** | Отсортированные по разделам журналы |
| **Средняя колонка** | Список событий выбранной категории |
| **Правая колонка** | Список доступных действий с выбранным журналом |
| **Нижняя панель** | Подробные сведения о конкретной записи |

### Разделы журналов

Всего разделов четыре:
1. Настраиваемые события
2. **Журналы Windows** (наибольший интерес!)
3. Журналы приложений и служб
4. Подписки

### Категории журналов Windows

| Категория | Описание |
| :--- | :--- |
| **Система** | События, сгенерированные системными компонентами (драйверами и модулями Windows) |
| **Приложения** | Записи, созданные различными программами |
| **Безопасность** | Сведения о безопасности (входы в аккаунты, управление правами, запуск процессов и т.д.) |
| **Установка** (доп.) | События, связанные с установкой обновлений и ПО |
| **Перенаправленные события** (доп.) | События, полученные с других компьютеров |

### Очистка журнала

**Способ 1 (ручной):**
1. Зайти в журнал событий
2. Кликнуть ПКМ по очищаемому журналу
3. Выбрать «Очистить журнал»

**Способ 2 (командная строка от имени администратора):**
```cmd
wevtutil cl "System"
wevtutil cl "Application"
wevtutil cl "Security"
```
