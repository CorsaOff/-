# Диагностика работоспособности персональных компьютеров
### Цель: проводить диагностику компьютера с помощью программ-диагностик.

PC Wizard - программа, которая предоставляет обширную информацию обо всех установленных на компьютере компонентах: память, материнская плата, устройства для хранения и записи информации, видеоподсистема, сетевые устройства, модемы, принтеры и т.д., включая разнообразные данные об операционной системе - версию системы, установленные шрифты, библиотеки, WinSock, активные процессы, имеющиеся модули и сервисы и т.п.
Кроме этого, PC Wizard позволяет протестировать быстродействие процессора, жестких дисков и CD/DVD-привода.
Поддерживаются 32- и 64-битные ОС

Порядок работы:
1.	Запустите виртуальную машину Windows 7 (которая осталась у вас от ассемблера)
2.	Перенесите туда файл из папки Занятие «pc-wizard_2014.2.14-setup.exe» (так как программа не совместима с Windows 10)
3. Установить программу PC Wizard.
4.  Запустить данную программу.
5. Слева предложены различные опции  программы, выбираем ТЕСТ

# Средство диагностики DirectX
Еще одно замечательное средство диагностики работоспособности компьютера называется Средство диагностики DirectX. 
Данная программа отображает сведения о компонентах и драйверах интерфейса Microsoft DirectX. Программа позволяет 
проверить работу аудио- и видеокарты, а также установить связь с мультимедийными службами. С помощью средства диагностики 
также можно отключить некоторые средства аппаратного ускорения.


## Ответы на контролльные вопросы

### 1. Причины возникновения проблем с совместимостью ПО
Основные «виновники» того, что программы не хотят работать вместе или в определенной среде:
Разные версии ОС: Программа написана под Windows 11, а запускается на Windows 7 (или наоборот).
Различия в архитектуре: Например, конфликт между 32-битными (x86) и 64-битными (x64) системами или переход на ARM-процессоры.
Зависимость от библиотек: Программе нужны конкретные версии сторонних библиотек (DLL, .NET Framework, Python-пакеты), которых нет в системе или они заменены более новыми.
Обновления ПО: Разработчики обновили API (интерфейс взаимодействия), и старый софт перестал «понимать» новый.
Специфическое железо: Использование нестандартных драйверов или жесткая привязка к определенным мощностям видеокарты/процессора.
Несоблюдение стандартов: Когда разработчики используют закрытые протоколы вместо общепринятых.
### 2. Методы уменьшения проблем с совместимостью ПО
Чтобы все работало гладко, используют следующие подходы:
Виртуализация и контейнеризация: Использование Docker или виртуальных машин (VirtualBox, VMware). Это позволяет запустить программу в «идеальной» для неё среде, которая не зависит от основной ОС.
Режимы совместимости: Встроенные средства ОС (например, «Запуск в режиме совместимости с Windows XP»).
Использование стандартов: Применение открытых форматов данных (JSON, XML) и стандартных API, которые не меняются годами.
Обратная совместимость: Когда разработчики новой версии софта намеренно оставляют поддержку старых функций.
Эмуляция: Использование специальных программ-слоев (например, Wine для запуска Windows-программ на Linux).
Статическая линкация: Включение всех нужных библиотек прямо в файл программы, чтобы она не искала их в системе.
